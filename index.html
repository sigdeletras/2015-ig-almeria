Información Geográfica en Internet (portales)

Desde hace unos años las administraciones públicas (directivas, leyes y normativas) publican sus repositorios y servicios de información geográfica en Internet.

Estos datos pueden ser agrupados en :

  • Archivos
    • Existe diversidad de formatos
    • Pueden ser descargados
    • Son guardados en local
    • Es necesario tratarlos con software geomático (BBDDG, Sistema de Información Geográfica o CAD)
    • Pueden ser modificados
    • Permiten simbología
    • Si los tiene, hay que tener en cuenta sus metadatos (ej. sistema de referencia de coordenadas, atributos, linaje, etc.)
  • Servicios
    • Basados en estándares de acceso. (WMS, WFS...) o no (Tiles)
    • Consumidos desde la Web
    • Simbología predefinida
    • En servicios OGC metadatos incluidos
    • Transformación de coordenas automática (servicios OGC)

Formatos de datos

Fuente Wikipedia

  • Shapefile (SHP). Formato multiarchivo propietario (ESRI). Formato SIG por excelencia.

Estructura ficheros de un archivo shape. ARCGIS Resource center

  • GeoJSON. GeoJSON is an open standard format for encoding collections of simple geographical features along with their non-spatial attributes using JavaScript Object Notation.

    • Ejemplo de fichero GeoJSON

  • KML. KML (del acrónimo en inglés Keyhole Markup Language) es un lenguaje de marcado basado en XML para representar datos geográficos en tres dimensiones. Fue desarrollado para ser manejado con Keyhole LT, precursor de Google Earth (Google adquirió Keyhole LT en octubre de 2004).

    <?xml version="1.0" encoding="UTF-8"?>
        <kml xmlns="http://www.opengis.net/kml/2.2">
          <Document>
            <Placemark>
              <name>New York City</name>
              <description>New York City</description>
              <Point>
                <coordinates>-74.006393,40.714172,0</coordinates>
              </Point>
            </Placemark>
          </Document>
        </kml>

  • GPX. GPX es un esquema XML pensado para transferir datos GPS entre aplicaciones. Se puede usar para describir puntos (waypoints), recorridos (tracks), y rutas (routes).

  • CSV. Los archivos CSV son un tipo de documento en formato abierto sencillo para representar datos en forma de tabla, en las que las columnas se separan por comas y las filas por saltos de línea. Pueden incorporar información geográfica mediante columnas de coordenadas X e y o bien en formato WKT.
  • GML. Acrónimo inglés de Geography Markup Language (Lenguaje de Marcado Geográfico). Es un sublenguaje de XML descrito como una gramática en XML Schema para el modelaje, transporte y almacenamiento de información geográfica.

Inframunicipal

CDAU

Proyecto Callejero Digital de Andalucía Unificado. Conjunto de datos alfanuméricos y geográficos de las vías y los portales de todos los municipios de Andalucía. Permite usar acceder a los datos (portales, tramos y vías) mediante un visor pero también la descarga de datos por municipios. La dirección de descarga es http://www.callejerodeandalucia.es/descargas/. Los formatos de descarga son GML o SHP.

Ejemplo de datos CDAU. Arboleas

Catastro

Descarga mediante certificado electrónico o DNI-e de datos catastrales de los municios de España. Los datos para descargar están en formato shape. También pueden descargarse información alfanumérica en formato CAT. Enlace a la Sede de Catastro
